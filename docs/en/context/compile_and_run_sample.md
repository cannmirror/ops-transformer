# Compile and Run Sample

## Prerequisites

- To compile and execute operator APIs, ensure the basic environment has been set up, including driver, firmware, CANN software package, ops package, and so on.

- For operator API calling flow and compile/run operation details, refer to [Application Development (C&C++)](https://www.hiascend.com/en) in "Single Operator Invocation > Single Operator API Execution > Calling aclnn Interface Sample Code".

## Pre-Compilation Preparation

This chapter uses the co-located development and runtime environment scenario as an example, meaning the machine with the AI processor serves as both the development environment and the runtime environment. In this scenario, code development and code execution are on the same machine. Here, the **FlashAttentionScore operator** is used as an example. The calling logic, flow, and compilation scripts for other operators are roughly the same as FlashAttentionScore. Modify the API calling script (*.cpp) and compilation script (CMakeLists) according to actual situations.

- **Sample Code**

   The FlashAttentionScore operator implements self-attention computation:

    - When psetype=1, add first then multiply.
    - When psetype≠1, multiply first then add.

- Computation formula:

  The forward computation formula for attention is as follows:

    - When psetype=1, the formula is:
      $$
      attention\_out = Dropout(Softmax(Mask(scale*(pse+query*key^T), atten\_mask)), keep\_prob)*value
      $$

    - When psetype≠1, the formula is:
      $$
      attention\_out=Dropout(Softmax(Mask(scale*(query*key^T) + pse),atten\_mask),keep\_prob)*value
      $$

  You can obtain the calling sample code from the [aclnnFlashAttentionScore](../../../attention/flash_attention_score/docs/aclnnFlashAttentionScore_en.md) document and name the code file "**test\_flash_attention_score.cpp**".

- **CMakeLists File**

    The CMake file sample is as follows. Modify according to actual situations:

    ```cmake
    # Copyright (c) Huawei Technologies Co., Ltd. 2025. All rights reserved.
    
    # CMake lowest version requirement
    cmake_minimum_required(VERSION 3.14)
    
    # Set project name
    project(ACLNN_EXAMPLE)
    
    # Compile options
    add_compile_options(-std=c++11)
    
    # Set compilation options
    set(CMAKE_RUNTIME_OUTPUT_DIRECTORY  "./bin")
    set(CMAKE_CXX_FLAGS_DEBUG "-fPIC -O0 -g -Wall")
    set(CMAKE_CXX_FLAGS_RELEASE "-fPIC -O2 -Wall")
    
    # Set executable file name (such as opapi_test), and specify the directory of the operator file *.cpp to be run
    add_executable(opapi_test
                   test_flash_attention_score.cpp) 
    
    # Set ASCEND_PATH (CANN software package directory, modify according to actual path) and INCLUDE_BASE_DIR (header file directory)
    if(NOT "$ENV{ASCEND_HOME_PATH}" STREQUAL "")      
        set(ASCEND_PATH $ENV{ASCEND_HOME_PATH})
    else()
        set(ASCEND_PATH "/usr/local/Ascend/cann")
    endif()
    set(INCLUDE_BASE_DIR "${ASCEND_PATH}/include")
    include_directories(
        ${INCLUDE_BASE_DIR}
        ${INCLUDE_BASE_DIR}/aclnn
        ${INCLUDE_BASE_DIR}/aclnnop
        # For custom packages, add additional include paths as follows, where ${vendor_name} needs to be modified to the name specified when compiling the custom package, default setting is "custom"
        # ${ASCEND_PATH}/opp/vendors/${vendor_name}_transformer/op_api/include 
    )
    
    # Set linked library file paths
    target_link_libraries(opapi_test PRIVATE
                          ${ASCEND_PATH}/lib64/libascendcl.so
                          ${ASCEND_PATH}/lib64/libnnopbase.so
                          ${ASCEND_PATH}/lib64/libopapi_math.so
                          ${ASCEND_PATH}/lib64/libopapi_transformer.so
                          # For custom packages, replace libopapi_transformer.so with libcust_opapi.so, reference path as follows,
                          # where ${vendor_name} needs to be modified to the name specified when compiling the custom package, default setting is "custom"
                          # ${ASCEND_PATH}/opp/vendors/${vendor_name}_transformer/op_api/lib/libcust_opapi.so 
                          )
    
    # The executable file is in the bin directory under the CMakeLists file directory
    install(TARGETS opapi_test DESTINATION ${CMAKE_RUNTIME_OUTPUT_DIRECTORY})
    ```

    When compiling operator samples generated by this project, you need to additionally link the libopapi_math.so dynamic library, because operators call some L0 interfaces during implementation, and these interfaces are encapsulated in libopapi_math.so. Therefore, this dependency must be explicitly declared during the compilation linking phase.

    For operators that fuse collective communication and MatMul computation and run them in parallel, collectively called MC2 operators (communication-computation fusion operators), including AllGatherMatmul, AlltoAllAllGatherBatchMatMul, BatchMatMulReduceScatterAlltoAll, MatmulAllReduce, MatmulAllReduceAddRmsNorm, MatmulReduceScatter, and so on. When calling these operator APIs, multi-threading and HCCL (Huawei Collective Communication Library) are generally involved, so the CMake file needs to additionally import the following content, otherwise compilation will not succeed.

  ```cmake
  # Set linked library file paths
  find_package(Threads REQUIRED)
  target_link_libraries(opapi_test PRIVATE
                        ${ASCEND_PATH}/lib64/libascendcl.so
                        ${ASCEND_PATH}/lib64/libnnopbase.so
                        ${ASCEND_PATH}/lib64/libopapi_math.so
                        ${ASCEND_PATH}/lib64/libopapi_transformer.so
                        # For custom packages, replace libopapi_transformer.so with libcust_opapi.so, reference path as follows,
                        # where ${vendor_name} needs to be modified to the name specified when compiling the custom package, default setting is "custom"
                        # ${ASCEND_PATH}/opp/vendors/${vendor_name}_transformer/op_api/lib/libcust_opapi.so
                        -Wl,--no-as-needed
                        ${ASCEND_PATH}/lib64/libhccl.so      # Collective communication library file
                        ${ASCEND_PATH}/lib64/libhccl_fwk.so
                        ${CMAKE_THREAD_LIBS_INIT})           # Multi-threading dependency library file
  ```

  Here "find_package(Threads REQUIRED)" is a CMake command for finding the thread library, which can automatically link header files or indirectly dependent library files that the thread library depends on.
  
## Compile and Run

  1. Prepare the operator calling code (*.cpp) and compilation script (CMakeLists.txt) in advance.
  2. Configure environment variables. 

     After installing the CANN software, log in to the environment using the CANN runtime user and execute the following command to activate environment variables.

        ```bash
        source ${INSTALL_DIR}/set_env.sh
        ```

     Here ${INSTALL_DIR} is the file storage path after CANN software installation. Replace according to actual situations. 
  3. Compile and run.
        - Enter the directory where CMakeLists.txt is located, execute the following command to create a build directory for storing generated compilation files.
          
            ```bash
            mkdir -p build 
            ```
        
        - Enter the build directory, execute the cmake command to compile, then execute the make command to generate the executable file.
          
          ```bash
          cd build
          cmake ../ -DCMAKE_CXX_COMPILER=g++ -DCMAKE_SKIP_RPATH=TRUE
          make
          ```

          After successful compilation, the opapi\_test executable file will be generated in the bin folder under the build directory.
          
        - Enter the bin directory and run the executable file opapi\_test.
        
        ```bash
        cd bin
        ./opapi_test
        ```
        
        Using the FlashAttentionScore operator running result as an example, the result sample after running is as follows:

        If the execution result has an error and the expected result does not appear, you can use the aclGetRecentErrMsg interface to obtain specific error information.
        A sample of obtaining exception information when calling aclnnFlashAttentionScoreGetWorkspaceSize with an error is as follows:

        ```cpp
        // q is nullptr
        ret = aclnnFlashAttentionScoreGetWorkspaceSize(
          q, k, v, pse, dropMask, padding, attenmask, prefix, scaleValue,
          keepProb, preTokens, nextTokens, headNum, layOut, innerPrecise,
          sparseMode, softmaxMax, softmaxSum, softmaxOut, attentionOut, &workspaceSize, &executor);
        CHECK_RET(ret == ACL_SUCCESS, LOG_PRINT("aclnnFlashAttentionScoreGetWorkspaceSize failed. ERROR: %d.\n[ERROR msg]%s", ret, aclGetRecentErrMsg()); return ret);
        ```

        The above null pointer issue error information sample is as follows:

